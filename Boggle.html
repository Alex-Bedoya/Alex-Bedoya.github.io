Grid Word Finder

A group project that allows 1 or 2 users to play a word finder game (Boggle) with a 1 minute timer. Which ever player gets more points, wins the game.

The goal of this project was to make a word finder game that one or two players can play at the same time and compete. The users would have 60 seconds to find as many words as they can in a 4x4 grid of random letters. The longer the word that the user finds, the more points they get for that word. They cannot use a single word twice, and once a user gets a word, the program tells the opposing player that the other player scored x amount of points. Once the 60 seconds is over, a results page is shown to both players, showing all words found and the winner of the game.

The main purpose of this project was for the group to get used to using Websockets. We needed to implement asynchronous technology so that the game would start and end at the same time for both users, while also sending data to each user about when they scored points. We also had to implement a way so that the program and game would look good on mobile devices. We decided to user Blazor for this project to help implement automatic page updating and websockets.

With this project, we were allowed to either use a database full of valid words, or an API. Our group decided to use an API, which I was in charge of. Since I never used Blazor before and implemented an API, I struggled quite a bit on implementing it, especially so that the game would wait for the verification of the word before giving points.

I also helped with making the interface mobile compatible, which was a simple CSS change depending on the width of the screen.
