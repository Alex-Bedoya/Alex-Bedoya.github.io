Movie Mania

As an Android Developer, I created an application that provides users with the functionality to store, rank, and curate their own movie lists. This project was developed as the culminating assignment of my Android class and was subsequently published to the Google Play Store.
To challenge myself, I chose to incorporate new technologies into this project. Specifically, I utilized Firebase for user account management and real-time database storage. Additionally, I integrated a RESTful API to retrieve movie data for the application. Overall, this project demonstrates my ability to develop and publish functional Android applications that leverage cutting-edge technologies.

In response to a need for organized movie tracking, I developed an Android application that enables users to curate multiple lists of movies, including a Master List, Want to Watch List, Owned List, and Ranked List.

The Want to Watch List incorporates a unique feature where users can randomly select a movie from their list with the click of a button. Meanwhile, the Ranked List enables users to reorder their list through a simple click-and-drag interface or share their list with others.

To add new movies to the Master List, users can utilize the search functionality accessed through the "+" button. Each movie in the Master List contains detailed information, such as title, release year, rating, runtime, average user score, director, composer, trailer link, overview, actors, and genres. Users can also edit movie details, including which lists the movie is in, user notes or comments, and a user rating out of five stars. Overall, this application showcases my proficiency in Android development and ability to create user-friendly interfaces.


As a developer on this project, I encountered a challenge in utilizing a RESTful API from "The Movie Database" to retrieve all necessary movie details. While the API allowed me to retrieve all data in a single call, parsing through the immense amount of data proved to be difficult.

One specific example of this was extracting the movie rating, which required parsing through all available ratings for each release and country. To achieve this, I iterated through each release in the US with an actual release date and identified the one with an "actual certification" to save as the rating.

Similarly, extracting the Director and Composer for the movie required looping through the entire crew list to identify individuals whose "job" was listed as "Director" or "Original Music Composer" and saving their names. These solutions demonstrate my ability to parse complex data structures and extract necessary information efficiently.


In developing this project, I utilized Firebase Database and RecyclerView to create a Master List feature. To achieve this, I first retrieved the database reference that points to the Master List of the current user.

Next, I added a database listener that triggers whenever data changes occur, and subsequently clears the current movie list and loads it with updated data. Once the data is loaded, I then sorted it in alphabetical order and displayed it on the RecyclerView.

Firebase Database stores data using JSON objects, which greatly simplifies data retrieval and makes it well-suited for scalable applications. This implementation showcases my proficiency in using Firebase Database and RecyclerView to create dynamic and efficient features.

Firebase proved to be a valuable tool for this Android application due to its ability to seamlessly integrate user account creation using email and password authentication. Firebase also handles password saving and security, which eliminated the need for me to implement these features.

One of the main advantages of using Firebase was the ability to create a separate dataset for each user, allowing for efficient loading and retrieval of data specific to each user. Additionally, user data can be saved in their account, making it easily accessible from multiple devices.


To enable click-and-drag functionality for the Ranked List feature, I utilized an ItemTouchHelper. This helper is triggered when the user clicks, holds, and drags an element on the screen.

Once the item is released, the onMove function is called, which notifies the collection to reorder based on the changes made by the user. Finally, I notify the Ranked List RecyclerView to update its display to reflect the newly reordered list.
