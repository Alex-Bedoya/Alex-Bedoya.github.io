Hangman Game

A solo project that allows the user to play hangman.

The goal of this project was to get familiar with the interactions between local and server side logic and storage of information. We were required to use PHP. The things that needed to get done with this Hangman game were to have a login system, so that users could have scores associated with their username, the actual Hangman game, that pulls a random word from a database and validates the characters that the user enters for guesses, and a leaderboard to display high scores of users who completed the game.

Some of the specifics of this assignment were that the app would need to store the passwords of the user hashed with a salt, and store that result in the database, so that the database doesn't actually have the real password. We also had to make sure the website that it was hosted on gave us a secure connection and supported SSL/TLS. I personally used Inifinty free as the host for the website, as it provided a good database to use with the site and also provided support for SSL/TLS.

Since this was more of a solo project for the class, I pretty much did the whole thing myself, with some help and suggestions from my group.
One of the hardest things for me to overcome in this project was figuring out how to presistently play the game even though the page reloaded every time the user guessed a letter. What I had to really utulize was session and the "isset()" function to store everything I needed and to check if those things are already in use or not. My biggest contribution to the group I think was the isset function as that helped my team figure out how to manipulate the session variables.

I also really liked figuring out how to properly print out the results page as a table.
