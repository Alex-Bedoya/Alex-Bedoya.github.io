Banking App

A group project that allows a user to have a Checking, Savings, and Other account to deposit, withdraw, and transfer money between each account.

The goal of this project was to get started working on a larger scale project with a team. This Banking app required us to let a user make an account, login to that account, and able to view their bank account. The user then was able to see 3 different sub-accounts inside their main bank account, which were a Checkings account, Savings Account, and an Other account. The user then would be able to deposit or withdraw money into each account, and was able to transfer money between each sub-account. We were also required to show a full transaction history of an individual sub-account, or a history of all accounts.

Some of the specifics of the assignment were that we were required to use .NET for the server end and use some sort of database to store all the data. We used Razor pages and used Microsoft SQL to store all the data we needed, which was usernames, salts, hashed passwords, account information, and transaction details. The last thing we had to do to help us learn more and get used to being on a team was to use a single code repo so that the entire group shares the same project, and we decided to use GitHub.

To make things easier, we each decided to do a seperate page for the project so that there would be less conflicts with logic and Github commits.
I was required to take care of the Transaction history page where it shows the user a history of an individual sub-account or all sub-accounts. Because we were using Razor pages, it was pretty easy to figure out how to properly print and format a table that shows all the transactions for a particular account.

One of the main things that I learned from this was how we access the database by using a data access class and call data very similar to using SQL, but not actually using any SQL. I wrote the two functions to get all transactions for a sub-account or all transactions for the whole account.
